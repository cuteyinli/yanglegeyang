# 数据库总览

## 技术选型

- **主数据库**：MySQL 8.0（生产环境） / SQLite（开发环境）
- **缓存层**：Redis 7.x（排行榜、会话缓存）
- **ORM**：SQLAlchemy 2.0

## 数据表设计

### 1. 用户表 `users`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT, PK, AUTO_INCREMENT | 用户ID |
| openid | VARCHAR(64), UNIQUE | 微信 OpenID |
| nickname | VARCHAR(64) | 昵称 |
| avatar_url | VARCHAR(256) | 头像地址 |
| created_at | DATETIME | 注册时间 |
| updated_at | DATETIME | 更新时间 |

### 2. 关卡配置表 `levels`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT, PK, AUTO_INCREMENT | 关卡ID |
| level_num | INT, UNIQUE | 关卡编号 |
| layer_count | INT | 卡片层数 |
| card_types | INT | 卡片种类数 |
| card_count | INT | 卡片总数（必须是3的倍数） |
| layout_data | JSON | 卡片布局数据（位置、层级） |
| difficulty | TINYINT | 难度等级 1-5 |
| is_daily | BOOLEAN | 是否为每日挑战关卡 |
| created_at | DATETIME | 创建时间 |

### 3. 游戏记录表 `game_records`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT, PK, AUTO_INCREMENT | 记录ID |
| user_id | INT, FK → users.id | 用户ID |
| level_id | INT, FK → levels.id | 关卡ID |
| status | TINYINT | 状态：0=进行中 1=通关 2=失败 |
| score | INT | 得分 |
| duration | INT | 用时（秒） |
| steps | INT | 操作步数 |
| started_at | DATETIME | 开始时间 |
| finished_at | DATETIME | 结束时间 |

### 4. 每日挑战表 `daily_challenges`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT, PK, AUTO_INCREMENT | 记录ID |
| date | DATE, UNIQUE | 日期 |
| level_id | INT, FK → levels.id | 对应关卡ID |
| total_players | INT | 参与人数 |
| pass_players | INT | 通关人数 |

## 索引设计

```sql
-- 用户表
CREATE UNIQUE INDEX idx_users_openid ON users(openid);

-- 游戏记录表
CREATE INDEX idx_records_user_id ON game_records(user_id);
CREATE INDEX idx_records_level_id ON game_records(level_id);
CREATE INDEX idx_records_score ON game_records(score DESC);

-- 每日挑战表
CREATE UNIQUE INDEX idx_daily_date ON daily_challenges(date);
```

## Redis 缓存设计

### 1. 排行榜（Sorted Set）

```
Key:    rank:global
Type:   ZSET
Score:  用户最高分
Member: user_id

示例：
ZADD rank:global 9800 1001
ZREVRANGE rank:global 0 99 WITHSCORES   # 获取 Top 100
ZREVRANK rank:global 1001               # 获取用户排名
```

### 2. 每日挑战排行（Sorted Set）

```
Key:    rank:daily:{date}
Type:   ZSET
TTL:    48小时

示例：
ZADD rank:daily:2026-04-22 156 1001     # Score = 用时（秒），越小越好
```

### 3. 用户会话缓存（String）

```
Key:    session:{token}
Type:   STRING
Value:  JSON { user_id, openid }
TTL:    7天
```

## ER 关系图

```
users (1) ──→ (N) game_records
levels (1) ──→ (N) game_records
levels (1) ──→ (1) daily_challenges
```

## 数据库迁移策略

- 开发阶段使用 SQLite，零配置即可运行
- 上线切换 MySQL，只需修改 `config.py` 中的数据库连接字符串
- 使用 Alembic 管理数据库版本迁移
