# 游戏界面比例调整指南

所有界面参数均为**屏幕比例值**（0~1），运行时根据设备实际屏幕尺寸动态计算像素值。修改对应常量即可调整布局，无需改动任何逻辑代码。

## 界面纵向布局总览

```
  0%  ┌──────────────────────┐
      │     标题区            │  TITLE_TOP ~ BOARD_TOP
 10%  ├──────────────────────┤
      │                      │
      │     卡牌区            │  BOARD_TOP ~ BOARD_BOTTOM
      │                      │
 65%  ├──────────────────────┤
      │     （间距）           │
 82%  ├──────────────────────┤
      │     槽位区            │  SLOT_TOP
 90%  ├──────────────────────┤
      │     道具区            │  PROP_TOP
100%  └──────────────────────┘
```

## 参数说明

以下参数定义在两个文件中：

- `client/scenes/game/cardRender.js` — 卡牌区、槽位区、道具区
- `client/scenes/game.js` — 标题区、结束覆盖层

### 1. 卡牌可用区域（cardRender.js）

控制卡牌能放置的矩形区域边界。

| 参数 | 当前值 | 基准 | 说明 |
|------|--------|------|------|
| `BOARD_TOP` | 0.10 | 屏幕高度 | 卡牌区顶部，标题下方 |
| `BOARD_BOTTOM` | 0.65 | 屏幕高度 | 卡牌区底部 |
| `BOARD_SIDE` | 0.07 | 屏幕宽度 | 卡牌区左右边距 |

**调整示例：**
- 想让卡牌区更高 → 增大 `BOARD_BOTTOM`（如 0.70）
- 想让卡牌区左右更窄 → 减小 `BOARD_SIDE`（如 0.04）
- 想让卡牌区从更高开始 → 减小 `BOARD_TOP`（如 0.08）

### 2. 卡牌绘制参数（cardRender.js）

控制单张卡牌的外观。

| 参数 | 当前值 | 基准 | 说明 |
|------|--------|------|------|
| `CARD_SIZE_SCALE` | 0.12 | 屏幕宽度 | 卡牌尺寸（宽=高），所有关卡统一 |
| `CARD_3D_DEPTH` | 0.1 | 卡牌尺寸 | 3D 侧面厚度 |
| `CARD_RADIUS` | 8 | px | 卡牌圆角半径 |
| `CARD_ICON_PAD` | 0.07 | 卡牌尺寸 | 图标距卡牌边缘的内边距 |
| `CARD_FONT_SCALE` | 0.45 | 卡牌尺寸 | 卡牌文字字号（无图片时的兑底显示） |
| `CARD_BLOCKED_THR` | 0.1 | 卡牌面积 | 遮挡判定阈值，重叠面积超过此比例则视为被遮挡 |

**调整示例：**
- 想让卡牌更大 → 增大 `CARD_SIZE_SCALE`（如 0.15），注意可能导致卡牌区放不下
- 想去掉 3D 效果 → 设 `CARD_3D_DEPTH` 为 0
- 想让图标更大 → 减小 `CARD_ICON_PAD`（如 0.03）
- 想让卡牌更圆润 → 增大 `CARD_RADIUS`（如 12）
- 想让遮挡判定更严格（更少重叠就算遮挡）→ 减小 `CARD_BLOCKED_THR`（如 0.05）

### 3. 槽位区参数（cardRender.js）

控制底部 7 格收集槽位的位置和外观。

| 参数 | 当前值 | 基准 | 说明 |
|------|--------|------|------|
| `SLOT_TOP` | 0.82 | 屏幕高度 | 槽位区顶部位置 |
| `SLOT_SIZE_SCALE` | 0.11 | 屏幕宽度 | 单个槽位的尺寸 |
| `SLOT_GAP_SCALE` | 0.015 | 屏幕宽度 | 槽位之间的间距 |
| `SLOT_PAD_SCALE` | 0.02 | 屏幕宽度 | 槽位背景的内边距 |
| `SLOT_ICON_PAD` | 0.06 | 槽位尺寸 | 槽位内图标的内边距 |
| `SLOT_FONT_SCALE` | 0.45 | 槽位尺寸 | 槽位文字字号 |

**调整示例：**
- 想让槽位整体上移 → 减小 `SLOT_TOP`（如 0.78）
- 想让槽位更大 → 增大 `SLOT_SIZE_SCALE`（如 0.13），同时可能需要减小 `SLOT_GAP_SCALE`
- 7 个槽位的总宽度 ≈ `7 × (SLOT_SIZE_SCALE + SLOT_GAP_SCALE)` 的屏幕宽度，注意不要超出屏幕

### 4. 道具区参数（cardRender.js）

控制底部 3 个道具按钮的位置和外观。

| 参数 | 当前值 | 基准 | 说明 |
|------|--------|------|------|
| `PROP_TOP` | 0.90 | 屏幕高度 | 道具区顶部位置 |
| `PROP_SIZE_SCALE` | 0.20 | 屏幕宽度 | 道具按钮尺寸（正方形） |
| `PROP_GAP_SCALE` | 0.08 | 屏幕宽度 | 道具按钮之间的间距 |
| `PROP_COUNT` | 3 | — | 道具按钮数量 |
| `PROP_RADIUS` | 10 | px | 道具按钮圆角半径 |
| `PROP_FONT_SCALE` | 0.28 | 按钮尺寸 | 道具按钮文字字号 |
| `PROP_LABEL_Y` | 0.72 | 按钮尺寸 | 文字纵向位置（从按钮顶部算起） |

**调整示例：**
- 想让道具按钮更小 → 减小 `PROP_SIZE_SCALE`（如 0.10）
- 想让道具按钮更紧凑 → 减小 `PROP_GAP_SCALE`（如 0.04）
- 想让按钮更圆润 → 增大 `PROP_RADIUS`（如 16）
- 想让文字居中偏上 → 减小 `PROP_LABEL_Y`（如 0.50）

### 5. 标题区参数（game.js）

控制关卡标题和返回按钮。

| 参数 | 当前值 | 基准 | 说明 |
|------|--------|------|------|
| `TITLE_FONT_SCALE` | 0.055 | 屏幕宽度 | 关卡标题字号 |
| `TITLE_TOP` | 0.02 | 屏幕高度 | 标题顶部位置 |

### 6. 结束覆盖层参数（game.js）

控制游戏结束/通关时的提示文字位置。

| 参数 | 当前值 | 基准 | 说明 |
|------|--------|------|------|
| `END_BIG_FONT` | 0.10 | 屏幕宽度 | 大标题字号（"恭喜通关"/"游戏结束"） |
| `END_BIG_Y` | 0.45 | 屏幕高度 | 大标题纵向位置 |
| `END_SMALL_FONT` | 0.05 | 屏幕宽度 | 小提示字号（"点击屏幕..."） |
| `END_SMALL_Y` | 0.52 | 屏幕高度 | 小提示纵向位置 |

## 调整注意事项

1. **区域不要重叠**：确保 `BOARD_BOTTOM` < `SLOT_TOP` < `PROP_TOP`，各区域之间留出间距
2. **槽位总宽度**：`7 × (SLOT_SIZE_SCALE + SLOT_GAP_SCALE)` 不要超过 `1 - 2 × SLOT_PAD_SCALE`
3. **卡牌区容量**：增大 `CARD_SIZE_SCALE` 后，关卡配置中的卡牌网格可能放不下，需要同步调整关卡布局
4. **道具区底部**：`PROP_TOP + PROP_SIZE_SCALE` 不要超过 1.0，否则按钮会被截断
5. **修改后清缓存**：在微信开发者工具中修改参数后，建议清除全部缓存后重新编译
