# 关卡设计指南

本文档说明如何新增和配置关卡。所有关卡文件位于 `client/scenes/game/levels/` 目录下。

## 快速上手：新增一个关卡

1. 在 `levels/` 下新建文件，如 `level3.js`
2. 按下方格式编写配置并 `module.exports` 导出
3. 在 `levels.js` 的数组中 `require` 该文件（数组顺序 = 游戏关卡顺序）

```js
// levels.js
const LEVELS = [
  require('./levels/level1'),
  require('./levels/level2'),
  require('./levels/level3'),  // ← 新增
]
```

---

## 配置结构总览

```
关卡
├── title        关卡标题（显示在游戏顶部）
├── iconTypes    使用的图标种类数
└── regions[]    区域数组（支持多区域布局）
    ├── x, y     区域左上角位置（相对卡牌可用区域 0~1）
    └── layers[] 层数组
        ├── layer       层级编号（0 = 最底层，越大越靠上）
        ├── gapRatio    卡牌间距 = 卡牌尺寸 × gapRatio
        ├── offsetCol   整层水平偏移（网格单位，支持小数）
        ├── offsetRow   整层垂直偏移（网格单位，支持小数）
        └── cards[]     卡牌位置数组 { col, row }
```

---

## 字段详解

### 关卡级

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | string | 关卡标题，显示在游戏界面顶部 |
| `iconTypes` | number | 使用几种不同的图标（从 18 种动物中随机挑选） |

### 区域级（regions）

每个关卡可以有多个区域，各区域独立定位。

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `x` | number | — | 区域左上角水平位置（0 = 可用区域最左，1 = 最右） |
| `y` | number | — | 区域左上角垂直位置（0 = 可用区域最上，1 = 最下） |
| `layers` | array | — | 该区域的卡牌层数据 |

> **坐标说明**：x, y 是相对于「卡牌可用区域」的比例，不是整个屏幕。可用区域由 cardRender.js 中的 `BOARD_TOP`、`BOARD_BOTTOM`、`BOARD_SIDE` 决定。

### 层级（layers）

每个区域内可以有多层卡牌叠放。

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `layer` | number | — | 层级编号。0 = 最底层，数字越大越靠上，高层遮挡低层 |
| `gapRatio` | number | 0.12 | 卡牌间距占卡牌尺寸的比例。0.1 = 紧密，0.9 = 稀疏 |
| `offsetCol` | number | 0 | 整层水平偏移（网格单位）。0.5 = 向右偏移半个格子 |
| `offsetRow` | number | 0 | 整层垂直偏移（网格单位）。0.5 = 向下偏移半个格子 |
| `cards` | array | — | 卡牌位置数组，每个元素 `{ col, row }` 指定网格坐标 |

## 坐标系图解

```
卡牌可用区域 (BOARD_TOP ~ BOARD_BOTTOM, BOARD_SIDE ~ 1-BOARD_SIDE)
┌──────────────────────────────────────┐
│(0,0)                          (1,0)  │
│                                      │
│     region { x: 0.12, y: 0.1 }      │
│     ┌─────────────────┐             │
│     │ col 0  1  2  3  │             │
│     │ row 0  ■  ■  ■  ■             │
│     │ row 1  ■  ■  ■  ■             │
│     │ row 2  ■  ■  ■  ■             │
│     └─────────────────┘             │
│                                      │
│(0,1)                          (1,1)  │
└──────────────────────────────────────┘
```

## 遮挡机制

卡牌的 `layer` 值决定遮挡关系：
- 高 layer 的卡牌渲染在上方，会遮挡低 layer 的卡牌
- 被遮挡的卡牌变灰且不可点击
- 判定标准：上层卡牌与下层卡牌的重叠面积超过下层面积的 10%（`CARD_BLOCKED_THR`）即视为被遮挡

**制造遮挡效果的方法**：
- 使用 `offsetCol`/`offsetRow` 让上层整体偏移，产生交错遮挡
- 例如 `offsetCol: 0.5, offsetRow: 0.5` 让上层向右下偏移半格，每张卡同时遮挡 4 张底层卡

---

## 卡牌数量约束

总卡牌数必须满足：

```
总卡牌数 = iconTypes × triplets × 3
```

其中 `triplets` 是每种图标出现的组数（自动向上取整），系统会自动填充卡池使之满足 3 的倍数。

**建议**：尽量让总卡牌数刚好等于 `iconTypes × N × 3`，避免多余卡牌。

## 现有关卡参考

### 第一关：新手引导

```
iconTypes: 3
布局：单区域，3 层 3×3
总计：27 = 3 × 3 × 3 ✓

特点：
- gapRatio: 0.9（间距大，卡牌分散，容易看清）
- 每层相同的 3×3 网格
- offsetRow 逐层递增（0 → 0.15 → 0.3），产生纵向交错
```

### 第二关：挑战开始

```
iconTypes: 8
布局：4 个区域
  主区域（中央）：5 层金字塔，93 张
  左侧小堆：3 层纵向，9 张
  右侧小堆：3 层纵向，9 张
  底部暗堆：单层横排，9 张
总计：120 = 8 × 5 × 3 ✓

特点：
- gapRatio: 0.1（间距小，卡牌紧密堆叠）
- 金字塔从底层 5×5 逐层缩小到顶层 3×4
- 多区域增加空间利用率和策略复杂度
```

## 设计技巧

1. **难度控制**
   - `iconTypes` 越多 → 同种图标越分散 → 更难消除
   - 层数越多 → 需要更多步骤揭开底层 → 更难
   - `gapRatio` 越小 → 遮挡越多 → 更难

2. **布局美观**
   - 用多个 region 创造空间层次感（主区域 + 小堆）
   - 左右对称的 region 视觉更平衡
   - 底部暗堆（单层）可作为"安全牌"缓冲

3. **偏移技巧**
   - `offsetCol: 0.5, offsetRow: 0.5` → 经典砖砌交错效果
   - 只用 `offsetRow` → 纵向滑动式遮挡
   - 较大的偏移值（如 1.0）→ 层间错开更大，遮挡减少

4. **验证清单**
   - [ ] 总卡牌数是 3 的倍数
   - [ ] `iconTypes × triplets × 3 ≥ 总卡牌数`
   - [ ] 所有 region 的卡牌在可用区域内不越界
   - [ ] 层级编号从 0 开始递增
   - [ ] 在微信开发者工具中实际运行测试布局效果
