- **平台**：微信小游戏

## 核心模块说明

### 1. 游戏渲染（Canvas 2D）

- 使用微信小游戏提供的 `wx.createCanvas()` 创建画布
- 通过 `requestAnimationFrame` 驱动游戏主循环
- 分层渲染：背景层 → 卡片层 → UI 层

### 2. 场景管理

| 场景 | 说明 |
|------|------|
| 主菜单 | 开始游戏、排行榜入口、设置 |
| 游戏主场景 | 卡片堆叠区域 + 底部 7 格槽位 |
| 结算场景 | 通关/失败结果展示、分享、重玩 |

### 3. 核心玩法逻辑

- **卡片堆叠**：多层卡片按规则堆叠，上层遮挡下层
- **点击消除**：点击可见卡片 → 移入底部槽位
- **三消判定**：槽位中凑齐 3 张相同卡片自动消除
- **胜负判定**：全部消除 = 通关；槽位满 7 张 = 失败

### 4. 与后端交互

- 使用 `wx.request()` 与后端 API 通信
- 登录流程：`wx.login()` 获取 code → 后端换取 openid
- 关卡数据从后端拉取，支持动态更新
- 游戏结果上报后端，用于排行榜统计

## 适配方案

- 使用逻辑分辨率，基于设备像素比（devicePixelRatio）进行缩放
- 以 750 × 1334 为设计基准，等比适配不同机型
